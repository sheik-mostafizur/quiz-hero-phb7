What is the difference forEach, map, and filter?

forEach: This method executes a callback function for each element in the array, but does not return anything. It is useful for looping over an array.

map: This method also executes a callback function for each element in the array, but returns a new array with the results of the callback function.

filter: This method executes a callback function for each element in the array, but returns a new array with only the elements that pass a certain condition.

What is the difference between Local Storage, Session Storage, and Cookies?

Local Storage: This is a mechanism that allows you to store up to 10 MB of data per domain that persists even after the browser is closed.

Session Storage: This is a mechanism that allows you to store up to 5 MB of data per domain that only lasts for the current session, i.e., until the browser (or tab) is closed.

Cookies: These are small pieces of data (up to 4 KB) that are sent by the server and stored by the browser. They are automatically sent back to the server on every HTTP request

What is Event loop and how to work it in JavaScript?

The event loop is a mechanism that allows JavaScript to execute asynchronous code in a single-threaded environment. It is responsible for executing the code, collecting and processing events, and executing queued sub-tasks1.

The event loop works by following these steps:

It executes the code in the main thread, which can be either synchronous or asynchronous. When an asynchronous function executes, it registers a callback function and returns immediately, without blocking the main thread. When the main thread is empty, the event loop dequeues and runs the oldest task from the macro-task queue. After each macro-task, it executes all the micro-tasks in the micro-task queue until it is empty2. It repeats this process until both queues are empty.
